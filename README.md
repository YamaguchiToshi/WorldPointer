# iPad対応Webアプリ仕様メモ

## 目的

車椅子を使用し、掲示物に近づくことが難しく、上肢の不随意運動により長い指し棒の操作が困難な児童生徒が、教室内の掲示物や黒板上の対象を「光のスポット」で指し示せるようにする。

## 想定環境

- iPad上のWebアプリとして動作
- iPadをプロジェクタまたは外部ディスプレイに接続
- 教室内での使用を想定
- 投影先は黒板、壁面、スクリーンなど
- 高精度な空間認識やAR処理は行わない
- 多少の投影歪みや位置ずれは許容する

## 基本動作

- アプリ起動時は黒背景を全画面表示する
- ポインティング操作が行われると、白いスポットを表示する
- スポットはポインティングデバイスの操作に応じて画面上を移動する
- 一定時間操作がなければスポットを自動的に非表示にする
- 非表示時は再び黒背景のみを表示する

## 入力方法

- iPadで利用可能なポインティング入力を想定する
  - タッチ
  - マウス
  - トラックボール
  - ジョイスティック型マウス
  - アクセシブルな外部ポインティングデバイス
- 将来的に以下も検討する
  - スイッチ入力
  - スキャン操作
  - 視線入力デバイスとの連携

## 表示仕様

- 背景色：黒
- ポインタ表示：白または任意色のスポット
- 初期状態では白い円形スポット
- スポットは小さな点ではなく、ある程度面積のある「スポットライト」風にする
- スポットサイズは調整可能にする
- スポット色は調整可能にする
- スポットの不透明度を調整可能にする

## 視覚エフェクト

PixiJSを用いて以下の表現を実装する。

- フェードイン
- フェードアウト
- Dissolve風の消失表現
- 円周の波状変形
- ゆるやかな脈動
- 柔らかい輪郭
- Glow表現
- 必要に応じて残像表現

## 操作補正

上肢の不随意運動に配慮し、将来的に以下を実装する。

- ポインタ移動のスムージング
- 移動速度の制限
- 急激な入力変化の抑制
- 停止時の安定化
- 入力感度調整
- dwellによる確定操作

## 使用ライブラリ

- PixiJS
  - 2Dグラフィックス描画
  - WebGLによる高速描画
  - スポットのアニメーション
  - フェードやGlowなどの視覚効果

## 実装形式

- HTML / CSS / JavaScript
- iPad Safariで動作するWebアプリ
- 可能であれば単一HTMLファイルでも動作可能にする
- 将来的にはPWA化を検討する
- フルスクリーン表示を前提とする

## UI設定項目

- スポットサイズ
- スポット色
- スポット透明度
- フェードアウトまでの時間
- 移動感度
- スムージング強度
- エフェクト有無
- 背景色

## 設計思想

- 精密な位置合わせよりも「どれを指しているか伝わること」を重視する
- 実世界を厳密に認識するのではなく、教室空間に対して大まかな注意誘導を行う
- レーザーポインタではなく、スポットライト的なポインティングを目指す
- 本人、教師、他児童が同じ対象を見る共同注意支援として設計する
- 身体動作を補うだけでなく、教室空間そのものをインタフェース化する
